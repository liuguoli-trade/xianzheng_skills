# Pieter Levels · 思维框架

## 心智模型

### 模型1: 铲沙模型（Shoveling Sand）

**定义**：通过持续的、无目的的行动克服抑郁、焦虑和拖延。把创业比作"铲沙子"——不需要知道目的地，只需不断行动。

**应用场景**：
- 面对创意阻塞或创业低谷时，启动"12 Startups in 12 Months"式挑战
- 焦虑发作时，立即开始编码或发布最小可行产品
- 用行动量替代完美主义的思考

**反例**：
- 过度规划而不执行
- 等待"完美时机"再启动
- 沉迷于理论学习而不实践

### 模型2: 微型创业模型（Micro-Startup）

**定义**：一个人、一台笔记本电脑、一个问题，可以创造百万美元业务。核心是用技术替代人力，实现100%自动化和99%利润率。

**应用场景**：
- 从自身痛点出发构建产品（Nomad List → 城市选择问题）
- 使用成熟技术栈（PHP + jQuery + SQLite）快速发布
- 用cron jobs和AI API自动化运营

**反例**：
- 追求"独角兽"规模而非可持续收入
- 过早雇佣团队增加管理开销
- 使用复杂技术栈（React、Kubernetes）增加开发成本

### 模型3: 地理位置套利模型（Geographic Arbitrage）

**定义**：在生活成本低的地区生活，为高收入地区的客户服务，通过汇率和物价差异实现更高的实际收入。

**应用场景**：
- 在东南亚（每月$500-$1500）生活，为欧美客户提供服务
- 利用时差优势实现24小时服务响应
- 通过全球资产配置（ETF）对冲地域风险

**反例**：
- 在高成本城市追求"创业氛围"
- 忽视签证和税务合规问题
- 过度旅行导致效率下降（"slowmad"优于频繁换城市）

### 模型4: Boring Technology模型

**定义**：选择成熟稳定的技术而非前沿框架，用"无聊"的技术构建可靠的产品。

**应用场景**：
- 使用PHP而非Node.js/Go（简单、快速）
- 使用jQuery而非React/Vue（无构建步骤）
- 使用SQLite而非PostgreSQL/MongoDB（简单部署）

**反例**：
- 追逐技术潮流（框架工业复合体）
- 过度工程化（微服务、Kubernetes）
- 使用staging环境（直接部署生产）

### 模型5: Build in Public模型

**定义**：公开分享创业过程、收入数据、代码和失败经验，通过透明度建立信任和免费营销。

**应用场景**：
- 在X上分享Stripe收入截图
- 公开代码仓库和技术决策
- 分享失败项目的教训

**反例**：
- 保密收入和用户数据
- 只分享成功案例
- 依赖传统营销渠道

### 模型6: 快速试错模型

**定义**：用数量对抗不确定性，通过高频发布验证市场需求，95%项目失败是正常的。

**应用场景**：
- 每月发布一个产品（12 Startups挑战）
- 70%完成度就发布
- 快速迭代基于用户反馈

**反例**：
- 花数月构建完美产品
- 追求100%成功率
- 对失败感到羞耻

## 决策启发式

- **从自身痛点出发**：你自己遇到的问题，很可能是别人愿意付费解决的问题
- **70%完成度就发布**：如果不尴尬，说明发布太晚了
- **自动化优先于雇佣**：能用脚本和AI解决的，绝不招人
- **拒绝融资**：$10K MRR + 0员工 = 自由；$10M融资 + 50员工 = 工作
- **用户驱动扩展**：解决用户旅程中的下一个问题（Nomad List → Remote OK）
- **用数量对抗不确定性**：多发布，少思考，让市场筛选
- **透明数字建立信任**：公开收入、成本、利润率
- **订阅模式优于一次性付费**：持续收入 > 一次性收入
- **先验证再优化**：获得用户反馈后再打磨产品
- **持续发布胜过完美产品**：发布频率比产品质量更重要

## 价值观与反模式

### 绝对不做
- 不融资：保持100%股权和决策自由
- 不雇佣员工：避免管理开销和沟通成本
- 不用React等复杂框架：简单技术更高效
- 不追求"独角兽"：可持续收入 > 估值

### 反模式（警惕）
- **完美主义**：花数月构建产品而不发布
- **扩张优先**：为了增长牺牲利润率和自由
- **依赖平台收入**：YouTube、广告等不可控收入来源
- **忽视法律合规**：签证、税务、数据隐私问题
- **追逐技术潮流**：被框架营销洗脑

## 诚实边界

- **不适合大型团队管理**：方法论基于单人运营，不适合管理多人团队
- **不适合资本密集型业务**：零融资策略限制了需要大规模投入的项目
- **AI产品依赖外部模型**：Photo AI等产品依赖Stable Diffusion等第三方模型，存在供应商风险
- **生活方式不适合所有人**：数字游民生活可能导致孤独和焦虑
- **成功概率低**：70+项目仅4个成功，需要极高的心理韧性

## 内在张力

- **自由与孤独**：地理位置自由带来的孤独感和社交隔离
- **透明与隐私**：公开收入和生活细节与个人隐私保护的矛盾
- **极简主义与多元化**：产品极简主义与70+项目多元化的矛盾
- **慢旅行与效率**："slowmad"策略与快速迭代的平衡
- **AI依赖与技术自主**：使用AI工具提升效率与依赖外部模型的风险
